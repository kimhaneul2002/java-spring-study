Ex) eat()처럼 부모클래스 타입으로 자식객체생성 <- 여러종류를 한번에 묶어서 처리하고싶을때
아닐 경우는
Ex) bark(),meow()처럼 자식클래스 타입으로 자식객체생성 <- 자식객체의 고유기능까지 다 써야할때


// 오버라이딩 - 부모클래스의 메서드를 자식클래스에서 새로 덮어쓰기


// 인터페이스
(내용없는 메서드 선언만) 약속목록들 인터페이스를 선언하고,
구현하는 클래스를 작성하고 오버라이딩필수


// 예외처리
Try - 예외발생코드
Catch - 에러발생시대응코드
finally - 에러 나든 안나든 마지막에 한번 실행코드



// 컬렉션(List,Map) - 여러개의 데이터를 한번에 담아서 관리하는 도구

— List컬렉션
1. names는 순서가 있는 리스트 형태의 자료구조를 다루는 변수다. 문자열만 저장가능
2. 그 List를 실제로 ArrayList 클래스로 구현한 객체를 만들겠다.
=> List<String> names = new ArrayList<>();


— Map컬렉션
1. scores는 이름표(key)와 값(value)을 한 쌍으로 저장하는 Map자료구조 다루는 변수다.
2. 실제로 HashMap 클래스로 구현한 객체를 만들겠다.
=> Map<String, Integer> scores = new HashMap<>();
